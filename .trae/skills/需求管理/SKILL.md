# 需求管理技能

## 功能描述

该技能支持敏捷开发工作流，主要功能包括：

### 1. 需求反向提取
- 从已实现的代码中分析并提取功能需求
- 识别功能模块和业务逻辑
- 生成结构化的需求文档

### 2. 需求管理流程
- **反向整理**: 根据已实现功能生成初始需求列表
- **人工调整**: 允许人工审核和调整需求内容
- **正向实现**: 根据调整后的需求指导功能开发

### 3. 需求文档生成
- 生成结构化的需求规格说明书
- 包含功能描述、业务规则、验收标准
- 支持用户故事格式

## 使用场景

**调用此技能当：**
- 用户需要从现有代码反向提取需求
- 需要对已实现功能进行需求文档化
- 进行需求评审和优先级排序
- 根据需求规划后续开发工作

## 工作流程

```
已实现功能 → 需求反向提取 → 人工审核调整 → 需求文档 → 功能开发/优化
```

## 输出格式

### 需求规格说明书结构
1. **需求编号**: REQ-XXX
2. **需求名称**: 简明描述
3. **功能描述**: 详细说明功能内容
4. **业务规则**: 功能的约束条件
5. **验收标准**: 功能验证条件
6. **优先级**: 高/中/低

## 示例

### 需求提取示例

**输入**: 股票投资助手系统代码

**输出**:
```
需求规格说明书
==============

1. REQ-001: 股票维护功能
   - 功能描述: 支持添加、删除、编辑股票信息
   - 业务规则: 股票代码格式为XXX.SZ或XXX.SS
   - 验收标准: 成功添加股票后显示在列表中

2. REQ-002: 智能选股功能
   - 功能描述: 根据策略筛选股票
   - 业务规则: 支持5种选股策略
   - 验收标准: 返回符合条件的股票列表

3. REQ-003: 模拟炒股功能
   - 功能描述: 模拟买入卖出股票
   - 业务规则: 初始资金10万元
   - 验收标准: 正确计算持仓和收益
```
