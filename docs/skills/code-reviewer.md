# code-reviewer 技能

自动化质量守门员，覆盖多语言规范、安全漏洞及可维护性审计。

## 触发条件

- 代码提交时自动触发
- 手动调用进行代码审查
- 任务完成后强制执行

## 审查维度

### 1. 代码规范

| 语言 | 检查项 | 标准 |
|------|--------|------|
| Java | 命名规范 | PascalCase 类名，camelCase 方法名 |
| Java | 行长度 | ≤ 120 字符 |
| Java | 注释规范 | Javadoc 公共 API，行注释逻辑 |
| JavaScript | 命名规范 | camelCase 变量，PascalCase 组件 |
| JavaScript | 行长度 | ≤ 100 字符 |
| TypeScript | 类型规范 | 禁止 any 类型 |

### 2. 安全漏洞

| 漏洞类型 | 检查项 | 修复建议 |
|----------|--------|----------|
| SQL 注入 | 使用参数化查询 | 使用 PreparedStatement |
| XSS | 未转义的用户输入 | 使用框架默认转义 |
| 敏感信息 | 硬编码密码/密钥 | 使用配置中心 |
| 权限绕过 | 未授权访问 | 添加权限注解 |

### 3. 可维护性

| 检查项 | 阈值 | 建议 |
|--------|------|------|
| 方法复杂度 | ≤ 10 | 拆分方法 |
| 类长度 | ≤ 500 行 | 拆分类 |
| 重复代码 | ≥ 3 次 | 抽取公共方法 |
| 圈复杂度 | ≤ 10 | 简化逻辑 |

### 4. 性能问题

| 检查项 | 问题 | 建议 |
|--------|------|------|
| N+1 查询 | 循环内查询 | 使用批量查询 |
| 大数据加载 | 一次加载全部 | 分页/懒加载 |
| 内存泄漏 | 未关闭资源 | 使用 try-with-resources |

## 执行命令

```bash
# Java 代码审查
mvn checkstyle:check

# 前端代码审查
cd frontend && npm run lint

# 安全扫描
mvn dependency:analyze
```

## 输出格式

```markdown
# 代码审查报告

## 审查结果: [通过/需修改/需重构]

### 问题列表

| 严重性 | 文件 | 行号 | 问题 | 修复建议 |
|--------|------|------|------|----------|
| 高 | UserService.java | 45 | SQL注入风险 | 使用参数化查询 |
| 中 | UserController.java | 30 | 方法过长 | 拆分方法 |
| 低 | UserMapper.xml | 20 | 命名不规范 | 改为 getUserById |

### 总体评价
- 代码规范: XX%
- 安全风险: X 个高危
- 可维护性: 良好/一般/差
```

## 约束规则

根据分层架构，以下调用是被禁止的：

```
✗ Controller → DAO (必须通过 Service)
✗ Service → 其他模块的 DAO
✗ 前端 → 直接访问数据库
```
