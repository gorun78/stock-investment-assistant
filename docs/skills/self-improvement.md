# self-improvement 技能

闭环自学习技能，收集错误案例并更新跨会话的项目记忆。

## 触发条件

- 任务完成后强制执行（复盘仪式）
- 错误修复后记录
- 新模式发现时

## 学习闭环

```
┌─────────────────────────────────────────┐
│                                         │
│   ┌─────────┐    ┌─────────┐           │
│   │ 收集错误 │───▶│ 分析根因 │           │
│   └─────────┘    └────┬────┘           │
│                       │                 │
│                       ▼                 │
│   ┌─────────┐    ┌─────────┐           │
│   │ 生成Skill│◀───│ 更新记忆 │           │
│   └─────────┘    └─────────┘           │
│                                         │
└─────────────────────────────────────────┘
```

## 记忆存储位置

```
docs/
├── session-learning/           # 会话学习记录
│   ├── 2026-03-19-xxx.md      # 按日期记录
│   └── README.md              # 学习索引
└── skills/                     # 技能库
    └── [skill-name].md        # 新增技能
```

## 记录模板

```markdown
# Session Learning - [日期]

## 任务概述
- **任务**: [任务描述]
- **模块**: [涉及模块]
- **结果**: [成功/失败]

## 遇到的问题

### 问题 1: [问题标题]
- **现象**: [问题描述]
- **根因**: [根本原因]
- **解决方案**: [解决方法]
- **预防措施**: [如何避免]

## 学到的经验

### 经验 1: [经验标题]
- **场景**: [适用场景]
- **最佳实践**: [推荐做法]
- **相关技能**: [关联技能]

## 新增/更新技能

- [ ] 新增技能: [技能名称]
- [ ] 更新技能: [技能名称]

## 后续行动

- [ ] [行动项 1]
- [ ] [行动项 2]
```

## 错误分类

| 错误类型 | 示例 | 预防措施 |
|----------|------|----------|
| 架构违规 | 跨层调用 | ArchUnit 检测 |
| 依赖冲突 | Bean 重名 | 模块隔离配置 |
| 性能问题 | N+1 查询 | 代码审查 |
| 安全漏洞 | SQL 注入 | 参数化查询 |
| 前端错误 | 组件 key 缺失 | ESLint 规则 |

## 知识库更新

### 1. 更新 AGENTS.md

当项目结构变化时：

```markdown
## 更新记录
- YYYY-MM-DD: 新增模块 XXX
- YYYY-MM-DD: 更新构建命令
```

### 2. 更新技能库

当发现新模式时：

```markdown
# 新技能: [技能名称]

## 触发条件
[描述触发条件]

## 执行步骤
1. [步骤 1]
2. [步骤 2]
```

### 3. 更新质量标准

当发现新问题时：

```markdown
## 新增检查项
- [检查项名称]: [检查标准]
```

## 复盘仪式

每项任务完成后必须执行：

```
1. 回顾任务目标
2. 总结执行过程
3. 识别改进点
4. 更新项目记忆
5. 分享学习成果
```
