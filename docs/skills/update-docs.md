# update-docs 技能

自动同步与自愈技能，任务完成后强制执行，确保文档与代码同步更新。

## 触发条件

- 任务完成后强制执行
- API 变更时自动触发
- 数据库变更时自动触发

## 同步范围

### 1. API 文档更新

| 变更类型 | 更新内容 | 目标文件 |
|----------|----------|----------|
| 新增接口 | API 定义文档 | docs/api-specs/ |
| 修改接口 | 参数/响应变更 | docs/api-specs/ |
| 删除接口 | 标记废弃 | docs/api-specs/ |

### 2. 数据库变更记录

| 变更类型 | 更新内容 | 目标文件 |
|----------|----------|----------|
| 新增表 | 表结构文档 | 各模块 docs/数据库设计文档.md |
| 修改表 | 字段变更记录 | database/migration/ |
| 新增索引 | 性能优化记录 | docs/architecture/ |

### 3. 架构决策记录

| 决策类型 | 记录内容 | 目标文件 |
|----------|----------|----------|
| 技术选型 | ADR 文档 | docs/architecture/ |
| 模块拆分 | 架构图更新 | docs/architecture/ |
| 依赖升级 | 变更说明 | CHANGELOG.md |

### 4. 功能清单更新

| 变更类型 | 更新内容 | 目标文件 |
|----------|----------|----------|
| 新增功能 | 功能描述 | 各模块 docs/功能清单.md |
| 修改功能 | 变更说明 | 各模块 docs/ |
| 废弃功能 | 废弃标记 | 各模块 docs/ |

## 执行流程

```
1. 检测代码变更
2. 识别变更类型
3. 匹配文档模板
4. 生成更新内容
5. 更新目标文档
6. 验证文档一致性
```

## 文档模板

### API 文档模板

```markdown
## [接口名称]

### 基本信息
- **URL**: `/api/v1/resource`
- **Method**: `GET/POST/PUT/DELETE`
- **描述**: [接口描述]

### 请求参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| ... | ... | ... | ... |

### 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 错误码
| 错误码 | 描述 |
|--------|------|
| ... | ... |
```

### 数据库变更模板

```markdown
## [变更标题]

### 变更信息
- **日期**: YYYY-MM-DD
- **版本**: vX.X.X
- **影响范围**: [模块名称]

### 变更内容
- [描述变更内容]

### SQL 脚本
```sql
-- 变更脚本
```

### 回滚脚本
```sql
-- 回滚脚本
```
```

## 验证规则

```
✓ API 文档与 Controller 方法一致
✓ 数据库文档与表结构一致
✓ 功能清单与代码实现一致
✓ 架构图与模块依赖一致
```
